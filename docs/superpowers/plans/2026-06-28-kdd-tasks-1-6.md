## Workstream A — Building-load + DER real-data validation (P0; tasks 1, 2)

**Files:**
- Create: `tools/validate_buildingload.py`, `data/buildingload_reference/` (cached ComStock/BDG2 profiles + `reference_bands.json`)
- Create: `tools/fetch_buildingload_reference.py` (idempotent downloader/characterizer)
- Modify: `tools/validate_calibration.py:595-661` (replace self-derived S5 bands)
- Test: `tests/test_validate_buildingload.py`

**Approach:**
- A1 (data sub-agent): download ComStock/EULP load profiles for the 5 shipped prototypes (Small/Medium/Large Office, Standalone/Stripmall Retail) across **multiple climate zones** (≥ CZ-5B Denver + 2–3 others), plus ~10–20 BDG2 office/retail meters. Normalize to 15-min or hourly, peak/area-normalize, persist a tidy parquet + a `reference_bands.json` of per-(archetype,size,CZ) load-shape stats (peak/off-peak ratio, weekday/weekend ratio, load factor, normalized 24h shape). Make it climate-zone-keyed so item #12 can reuse it.
- A2 (validator sub-agent): `validate_buildingload.py` generates the generator's CZ-5B load (peak_kw_scaling OFF), aligns to the matched ComStock/BDG2 profile, computes ASHRAE Guideline-14 metrics — **CV(RMSE) ≤ 30%, NMBE ≤ ±10%** — plus normalized-shape correlation, peak-hour error, load factor; emits a metrics table.
- A3 (bands sub-agent): replace `PNNL_EXPECTED_PEAK_OFFPEAK` / weekday-weekend bands (`validate_calibration.py:600-609`) with ranges derived from A1's real-data `reference_bands.json`; keep peak/off-peak as a coarse smoke test only; re-emit `docs/CALIBRATION_RESULTS.md`.

**Verification gate:** validator runs end-to-end; ≥3/4 archetypes pass for the *right* (real-data) reason or failures are explained as real model limits; full `pytest` green.

- `uv run pytest tests/test_reproducibility.py tests/test_determinism_stress.py` MUST stay green — synthetic populations bitwise-identical.

**Verification gate:** determinism tests green; `model_eval` shows arrival/dwell KS dropping toward ~0.03; `rare_consistent` arrival KS materially improved from 0.222; full `pytest` green.

**Approach:**

**Verification gate:** a real numeric transfer result with the TSTR-vs-TRTR gap quantified; smoke test green.
