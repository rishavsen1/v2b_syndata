§2Layer 1 — Descriptor expansion & knob resolution

Everything the DAG consumes is a knob: a typed, ranged, defaulted value in a flat bucket.name namespace. Resolution happens once, up front, and every resolved value remembers where it came from.

Priority chain

CLI --override  ≻  scenario YAML overrides  ≻  descriptor expansion  ≻  knobs.yaml default

Each resolved knob is a KnobValue(value, source); source ∈ explicit · descriptor:<name> · calibration:<provenance> · hand_specified:<population> · default — all recorded in the manifest's knob_resolution.

Two channels

Registry channel

  • Paths declared in configs/knobs.yaml (the 69 knobs below).
  • Typed: int / float / bool / categorical / simplex / vec2 / list / path / timestamp, each with range or choices.

Deep channel

  • user_behavior.region_distributions.<region>.<dist>.<param> leaves (calibrated fits).
  • Validated against DIST_PARAM_RANGES (e.g. arrival.mu ∈ [4, 22], dwell.lambda ∈ [0.01, 24]); out-of-range ⇒ KnobValidationError. Metadata leaves (n_samples, ks_fit_quality) are filtered out, not knobs.

§3Layer 2 — Seeding: hash-keyed, order-independent randomness

Every random draw in the pipeline comes from a named sub-stream derived from the single seed — keyed by a SHA-256 hash of the node name, never by spawn order.

stable_int(name)  = int.from_bytes(SHA256(name)[:4], "big")
node stream       = default_rng(SeedSequence(entropy=seed, spawn_key=(stable_int(node),)))
per-car stream    = default_rng(SeedSequence(entropy=seed, spawn_key=(stable_int(node), car_id)))
date-keyed stream = rng_for_car(seed, f"sessions:{day.isoformat()}", car_id)   # sessions day loop
  • Adding a node never shifts existing seeds — hash keys are position-free. (Python's salted hash() is banned.)
  • Auxiliary draws get their own streams (e.g. dirichlet:axes, weather_realization, weather_realization_solar/_dewpoint/_wind) so they can't perturb per-car streams.
  • Rejection sampling consumes RNG deterministically — retries advance the same stream, so results stay bit-stable.
  • Guarded by test_reproducibility.py and test_determinism_stress.py.

§5Subsystem deep dives

5a · Building load — the EnergyPlus pipeline

  1. Prototype: (archetype, size) → ASHRAE 90.1-2019 PNNL Denver-CZ5B IDF (OfficeSmall / OfficeMedium / OfficeLarge / RetailStripmall / RetailStandalone); mixed = office + retail runs averaged 50/50.
  2. Occupancy injection: the chosen ashrae_90_1_* weekly schedule replaces the IDF's BLDG_OCC_SCH* objects (weekday day-peaking office / evening retail; office weekends ≈ 0).
  3. Weather: EPW for tmyx_station (auto-downloaded, cached) perturbed in-memory: +temp offset, ×solar scale, +dew-point offset (RH recomputed via Magnus), ×wind scale. The same perturbed frame is exported to weather_data.csv and fed to PV — load, PV and exported weather can never disagree.
  4. Simulation: RunPeriod = sim window year, Timestep 4 (15-min), 8 end-use meters → eplusmtr.csv in joules.
  5. Parsing: kW = J / 900 000; flex = Cooling + Heating + Fans + WaterSystems, inflex = Interior/Exterior Lights + Interior/Exterior Equipment; reindexed to the canonical 15-min grid.
  6. Peak scaling: if peak_kw_scaling (default), flex+inflex rescaled so max(total) = peak_kw exactly; else raw magnitudes (weather-driven peaks).

5b · Sessions — distributions, copula, and the 5-constraint rejection

Per car, per day (weekdays, unless weekdays_only=false): appear with prob. φ (weekends: φ × weekend_activity_factor). On appearance, draw a session with up to 5 retries, each attempt atomic:

Draws

  • Arrival: TruncNorm (or 2-comp mixture) via inverse-CDF on a copula uniform.
  • Dwell: Weibull (or mixture), clipped; shares a Gaussian-copula pair with arrival when ρ ≥ 1e-9 (ρ fixed per car before the day loop — C4 region-stable).
  • Arrival SoC: shifted, clipped Beta.
  • Required SoC: calibrated Beta or TruncNorm(85, 5) fallback, floored at max(min_depart_soc, arrival + 0.01) and capped at max_allowed_soc.

Reject if any of

  • C12 — departure crosses midnight (no overnight).
  • Time bounds — outside the sim grid.
  • Non-overlap — collides with the car's prior session.
  • D6 — required ≤ arrival (structural floor).
  • D5 — energy unreachable: (required−arrival)/100 × capacity > max_rate × dwell.

All retries exhausted, or floor ≥ ceiling (arrived too charged) → the car-day is dropped. Enforcement by rejection, not clamping (DESIGN_NOTES §4).

5c · PV — PVWatts v5 on the same weather

  1. Resolve spec from der_catalog (preset or explicit kW); pv_type: none ⇒ all-zero series, zero I/O.
  2. Hourly perturbed EPW → 15-min grid (forward-fill); solar geometry evaluated at each tick's midpoint (+7.5 min).
  3. Isotropic transposition: POA = beam + sky-diffuse (1+cos β)/2 + ground-reflected · albedo (1−cos β)/2.
  4. Cell temp (NOCT): Tcell = Tdb + (POA/800)(NOCT−20); DC = dc_kW · (POA/1000)(1 + γ(Tcell−25)); AC = min(DC · derate, ac_kW).

Pure deterministic physics — no RNG. PV varies across samples only through the weather realization.

5d · Grid prices & DR events

  • grid_prices.csv: 15-min series from the tariff bundle — flat (single price, all off-peak) or TOU (peak price inside peak_window hours). Deterministic.
  • dr_events.csv: Lewis-thinned inhomogeneous Poisson. λ(t) = dr_lambda_base (events/hour, D63) × seasonal gate × weekday 3:1 × temperature ramp (0.1 below 80 °F → 3.0 above 100 °F, driven by the EPW's daily max dry-bulb) × time-of-day (peak 15–17 h, zero outside 12–20 h), capped by program monthly/seasonal maxima. Magnitude ~ U(dr_magnitude_kw_range); notification lead: CBP 24 h, BIP/ELRP 2 h. dr_program: none ⇒ empty file.

5e · Stationary battery

Specs-only row (capacity, power, η, SoC window, initial SoC) from preset or explicit knobs. Deliberately no dispatch schedule — dispatch is endogenous to the downstream V2B optimization, not exogenous data (PROJECT_TRACKER 2026-06-29 decision).

§6Layer 4 — Output noise (and how it differs from weather)

Weather layer — input-side

  • Perturbs the EPW before EnergyPlus/PV; exported weather matches.
  • Physically faithful: load stays exactly f(weather).
  • Presets (weather_profiles.yaml): none · slight (σ 1 °C / 2% solar / 0.5 °C dew / 5% wind) · moderate (2.5 / 5% / 1.5 / 10%) · strong (5 / 10% / 3 / 20%).
  • Never violates validation — variance is upstream of all constraints.

Noise layer — output-side

  • Perturbs the rendered CSVs after generation (measurement noise).
  • Can push sessions past constraints enforced during sampling → D5 flags.
  • Presets (noise_profiles.yaml): clean · light_noise · realistic_noise · adversarial · tmyx_stochastic · custom.

Guard rails inside the noise layer

  • C4 forward-shift bound — arrival jitter clamped so ≥ 30 min dwell survives and C12 (same-day) holds.
  • D6 clamp — jittered arrival SoC clamped to [car min, required − 0.1] so ordering survives.
  • Post-jitter D5 enforcement (_enforce_d5_post_jitter) — required SoC truncated to arrival + rate·dwell·1.04 headroom, floored at arrival + 0.01; irreparable sessions dropped; truncation/relaxation/drop counts recorded in manifest.noise.d5_enforcement.
Field-proven interaction: even with those guard rails, tmyx_stochastic-class jitter nudges ~50% of multi-sample batch units past the validator's strict D5 envelope (validator headroom 1.0 vs enforcement 1.04). For Monte-Carlo batches, the proven recipe is noise: clean + a weather profile: full sample diversity (weather re-sim + per-sample seed), zero hard errors — verified at 18,000 units / 0 errors on the campus10 run.

generate-multi — one run, N buildings

  1. Config YAML: globals (output_mode, dr_program, dr_incentive_per_kw, dr_penalty_per_kwh, default_policy) + buildings[], each with base_scenario, descriptor swaps, overrides, seed, optional per-building noise_profile / weather_profile / policy / multiplier.
  2. multiplier: N expands one entry into N distinct realizations (seeds base…base+N−1); building_id assigned sequentially across the fully-expanded list.
  3. Each building runs the full L0–L6 stack natively in an isolated tempdir, then converts to optimus tables.
  4. Output modes: shared (one CSV set, building_id column) or per-building (integer subfolders 0..N−1 — note: not a building-major tree; for parent/b1..bN run one batch per building).
  5. Writes multi_building_config.json: expanded specs + globals + per-building validation rollup — the regeneration recipe (--from-config reproduces byte-identically).

Unified batch — buildings × months × samples

  1. Tree: <out>/<MONTH>/<sample>/ (one optimus set per unit).
  2. Seed strategy: building.seed + seed_base + sample — space building base seeds ≥ samples_per_month or ranges collide.
  3. Weather realization: per building, its own weather_profile (else the batch default) → σ's; each (unit, building) draws temp/solar/dewpoint/wind offsets from streams weather_realization* seeded by the per-sample seed, pinned as explicit knob overrides. Explicit per-building offsets in the spec win. CLI --weather-sigma-c/--weather-solar-sigma override σ for every building (power-user).
  4. ProcessPool over units; batch_manifest.json records config, seed strategy, per-unit results and a batch-wide validation_summary.

Web tool

Flask + vanilla JS (tools/web/): descriptor/knob/scenario/DER catalog browsing, per-building cards with knob panels, PV/battery pickers, per-building weather noise, run-level output-mode toggle (shared vs per-building), single-shot generation with CSV previews, batch submission with live progress, ZIP download, distribution plots per building_id.

Post-run analysis (repo tooling)

tools/analyze_overnight.py (single building, slight-vs-moderate trees) and tools/analyze_campus.py (building-major b1..bN) stream every sample → per-sample metrics CSV + self-contained uncertainty-analysis HTML (CV rankings, P5–P95 bands, per-building box plots).

§10The reproducibility contract & the gotchas that bite

The contract: (scenario, seed) → byte-identical CSVs, when noise = clean, Dirichlet α's at their 1e6 off-defaults, and DER off-by-default preserved. Every stochastic feature added later (Dirichlet mixes, weather realizations, PV/battery) was designed to be exactly off at defaults so the historical bitstream is untouched. Every unit is regenerable from seed + config (--from-config).
GotchaReality
tmyx_stochastic in multi-sample batchesOutput jitter lands after D5 rejection → ~50% of units flag D5 at validation. Use clean + weather profile for valid Monte-Carlo variance.
Global --weather-profile vs per-buildingPer-building weather_profile: in the config wins; the CLI flag is only the fallback default — don't pass it when mixing profiles.
peak_kw_scaling defaulttrue ⇒ every sample's peak == descriptor peak_kw exactly (variance only in energy/shape). Set false for weather-driven peaks.
--output-mode per-buildingLeaf-level 0..N−1 folders under MONTH/sample — not a building-major tree. Building-major = one single-building batch per building.
Seed spacingBatch adds 0..samples−1 to each base seed; space bases ≥ samples_per_month (and ≥ multiplier).
cli validate on optimus dirsFails on schema (native-only). Read validation_summary from the batch/config manifests instead.
climate knobLabel only. The weather that matters is tmyx_station.
Occupancy vs sessionsOccupancy schedules shape building load only; EV sessions come from the population axes — deliberately decoupled.
weather_data.csv resolutionHourly (native EPW), everything else 15-min — resample before joining.
SoC "data"No public charger dataset records SoC. All SoC quantities are modeled priors (reconstructed from kWh delivered/requested) — never treat them as measured ground truth.
